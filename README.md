A plain-language, choose-your-own-path introduction to Section 22 of the JBNQA.

> **Proposed does not mean decided.**
>
> This project helps people explore how community knowledge, questions, and participation can shape what happens next.

## What this project does

- presents Section 22 as an interactive, easy-to-follow experience
- helps visitors understand how a proposal moves through notice, questions, review, and action
- centers Cree voices, land-based knowledge, and community participation
- uses clear language instead of legal jargon

## Experience overview

### Home path
A short interactive introduction where users choose responses and see how their decisions affect the process.

### Section 22 flow
A six-stage plain-language map of the process:
1. Notice
2. Questions
3. Knowledge
4. Hearing
5. Recommendation
6. Follow-up

## Project aesthetic

The site uses a warm editorial style with:

- bold headlines
- soft paper-like panels
- hand-drawn / illustrated imagery
- strong contrast and simple navigation
- ribbon-style callouts and emphasis text

## Tech stack

- **TypeScript**
- **Next.js / Vinext**
- **Cloudflare-ready deployment shape**
- **Optional D1 + Drizzle support**

## Getting started

```bash
npm run install:ci
npm run dev
```

## Available commands

- `npm run install:ci` — perform the one bounded lockfile install
- `npm run dev` — start the development server
- `npm run build` — build and validate the deployable artifact
- `npm run start` — start the built application
- `npm test` — build, validate, and verify the preview metadata
- `npm run validate:artifact` — recheck an existing artifact
- `npm run db:generate` — generate Drizzle migrations after schema changes

## Notes

This site is designed to be educational and accessible. It is not a legal interpretation or replacement for official guidance.
